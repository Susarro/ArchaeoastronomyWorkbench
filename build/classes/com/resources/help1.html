A Project is a homogeneus set of alignments to be analyzed so that we can get a pattern of its main astronomical declinations and be able to value its fortuitousness in relation to a an astronomical scheme based on solstices, equinoxes, cross-quarter festivals and lunar standstills [1]. This set can be a population of orientations corresponding to the main axis of certain monuments, or alignments due to some celestial bodys rising or setting on some conspicuous horizon landmark as seen from an megalithic or rock art site, or directions defined by some features of engravings or paintings. You can create a project choosing New project in Project menu or open an already existing one clicking Open project. In any case a tab pane will be shown with the given project name and two menus: File and Data analysis.

By means of File menu you will be able to add new alignments choosing New alignment, save the project or export it in comma separated values (CSV) format.

Data analysis provides two kinds of evaluations: the probability of fortuitous astronomical coincidences from a number of alignments  using two criteria: Rosenfeldt or Bernoulli, and the probability density estimation using a Gaussian kernel.

1.- File

1.1.- Add, delete and edit alignments.

New alignment option menu allows you to add a new alignment to the current project. This selection will show a dialog for entering its name and the geographical coordinates (latitude, longitude and elevation) of the observatory site and the foresight horizon point (Coordinates option checked) or the declination and altitude angles (Coordinates option unchecked). This setting will be converted to an astronomical declination value. It's important to mention that if the Coordinates option is checked the observatory coordinates (and foresight coordinates too) must be as exact as posible but this accuracy is not necessary when azimuth and altitude angles are entered. Finally the new alignment is added as a row in the alignments table.

If you select an alignment in the table you can open its context menu and select Delete to delete it or Edit to change some of its properties. 

1.2.- Save project

Save option menu allows to save the project in a XML (eXtensible Markup Language) file using the file extension *.awb.

1.3.- Export project

Export option menu allows to export the project in a CSV (Comma-Separated Values) file.

1.4.- Alignment operations

When you select a row in the alignments table a pane will apper to the right with two tab panes: Details, which shows detailed information of the aligment and Validation, wich provides you some interesting validation tools that will be described bellow.

Details tab pane shows detailed information of the alignment to that you can include an image. To get it you should click on Update button which will open a dialog with a list of the thumbnails of the images imported to the project and Import, OK and Cancel buttons. If you select Import a file chooser dialog will be opened to select an image file that will be copied to the image folder of your project and the corresponding thumbsnail will be added to the list. If you select one of these thumbsnails and click on OK this image will be linked to the alignment and shown in the details pane of the alignment. Delete button will remove the link to the image (but not the image from the project image folder).


Validation tab pane allows you to validate an alignment in relation to solstices, equinoxes, cross-quarter festivals and lunar standstills[2]. In the combo list Astronomical event you can choose the event to consider: winter solstice, summer solstice, Imbolc/Samain, Beltaine/Lugnasad, Southern Lunar major standstill, Northern Lunar major standstill, Southern Lunar minor standstill, Northern Lunar minor standstill or None. The year is also required to calculate the precise declination values. After selecting an astronomical event the color of a led on the right will change to green if the arror in azimuth is equal or less than 1º or to red otherwise. If the validation process is successful, i. e., the error value is not greater than 1º, the astronomical event identification is add to the comments field of the alignment. If the astronomical event selected is None the comments field will be erased.

2.- Analysis

This menu allows to perform to kinds of statistical analysis min the alignments population of the project: the probability of fortuitous astronomical coincidences from a number of alignments and get the probability density estimation using a Gaussian kernel.

2.1.- Probability of fortuitous astronomical coincidences from a number of alignments

The current project consists of alignments, some of them can be considered "coincidences" if they were validated using the Validation procedure described previously (2.1.4.). The probability of fortuitous astronomical coincidences can be calculated by means of two criteria: a Bernoulli process[3] or the procedure described by Rosenfeldt[4].

2.2.- Probability density estimation using a Gaussian kernel

This option allows you to get the estimation of the probability density function using a Gaussian kernel from the declination angles population of the current project. A dialog will be shown to enter the standard deviation of the gaussian kernel and a declination step because this density estimation is dicretized.
